# 🧪 六合彩公式实验室（重构版 v3）

本地运行的"测号公式回测 + 实战预测"工具。本次迭代的核心变化：

- ✅ **严格区分"真公式"与"假公式"**  
  真公式 = 只用上1/上2/上3 期的数据推导下一期 → 可以用来实战  
  假公式 = 引用了本期已开奖字段 → 只能解释历史，**不能预测**
- ✅ **新增"实战预测"页**：对下一期直接输出候选排行榜
- ✅ **单公式回测页直接给出下一期预测 + 推导路径**
- ✅ **公式构建器改为多块积木**，支持原子/二元/三元/条件/聚合/包装/**函数公式**层层组合
- ✅ **自动寻优只挖真公式**
- ✅ **新综合评分**：近100 40% + 近50 25% + 近20 15% + 稳定性 10% + (1-连黑惩罚) 10%
- ✅ **wrap49 归一化**贯穿所有映射
- 🆕 **函数公式系统**：内置 8 个通用函数（F_sum2, F_sum3, F_add_sub, F_absdiff, F_max3, F_min3, F_if_gt, F_mod49_sum3）+ 用户自定义。**函数内部藏 lag=0 的因子同样会被校验器抓到**，绕不过去。
- 🆕 **三期交叉模板底层**：最近 3 期的 7 个号码视为 3×7 矩阵，支持上下/左右/主对角/反对角/十字/九宫格/自定义路径取格子，可直接聚合成 wrap49 表达式。*UI 集成留在下一轮。*

> 本工具只用于数据研究和公式回测实验，不提供任何投注建议，也不保证预测命中率。

## 🔒 真公式 / 假公式 的严格定义

### 定义

对 AST 里每一个**因子节点** `{"factor": X, "lag": Y}`：
- 若所有因子的 `lag ∈ {1, 2, 3}` → **真公式**
- 只要有任意一个因子的 `lag == 0` 或未指定 → **假公式**

### 为什么

回测第 i 期时，真公式只读取 i-1/i-2/i-3 期的数据，这就**等价于**实战中对下一期 t+1 的预测只用 t/t-1/t-2 期数据。

反之，公式若引用"本期任何字段"（平1~平6 或 特码），那它就是用**答案本身**在"预测"答案 —— 这种公式的回测胜率是虚假的，不能在实战中使用。

### 在各个页面的表现

| 页面 | 真公式 | 假公式 |
|---|---|---|
| 公式构建器 | ✓ 绿色徽标，可保存、可预测 | ⚠ 红色警告，仍可保存但标记 |
| 单公式回测 | ✓ 下一期预测正常显示 | ❌ 下一期预测拒绝输出 |
| 自动寻优 | ✓ 只产出这类 | ❌ 完全不会出现 |
| 公式库 | 默认筛选只看这类 | 需手动切到"只看假公式" |
| 实战预测排行榜 | ✓ 唯一的来源 | ❌ 直接排除 |

## 📐 公式 AST 简介

```json
{
  "target": "一肖",
  "expr": {
    "op": "map_to_zodiac",
    "args": [
      {"op": "wrap49",
       "args": [
         {"op": "add",
          "args": [
            {"factor": "平1", "lag": 1},
            {"factor": "平2", "lag": 1}
          ]}
       ]}
    ]
  }
}
```

### 因子（`"factor"`）

**号码**：`平1 平2 平3 平4 平5 平6 特码`  
**聚合**（对该期 7 个号码做聚合）：  
`七码和` `六码和` `七码均值` `最大号` `最小号` `跨度` `七码尾和` `七码头和`

每个因子都必须带 `lag`（0=本期，1/2/3=上1/2/3 期）。号码类因子还能带 `attr`（头/尾/合/合尾/大小/单双/合单双/合大合小/段数/生肖/波色/五行/家禽野兽）。

### 运算（`"op"`）

**数值**：`add sub mul div abs mod floordiv avg max min`  
**归一**：`wrap49`（映射前自动调用）  
**位处理**：`tou wei hes he_wei digit_diff digit_prod`  
**比较（返回 0/1）**：`gt lt eq between same_wei same_wave same_zodiac`  
**条件**：`if_else`  
**映射**：`map_to_zodiac map_to_wei map_to_tou map_to_wuxing map_to_duan map_to_wave map_to_bs map_to_oe map_to_animal map_to_he_oe map_to_he_bs map_to_he_wei`  
**集合**：`to_numbers map_to_custom_set`

## 🆕 函数公式

把常用套路封装成可复用函数，不用每次手拼。

**内置 8 个**（位于 `data/function_formulas.json`，首次启动自动创建）：

| 函数 | 签名 | 含义 |
|---|---|---|
| `F_sum2` | `(a, b)` | `wrap49(a + b)` |
| `F_sum3` | `(a, b, c)` | `wrap49(a + b + c)` |
| `F_add_sub` | `(a, b, c)` | `wrap49(a + b - c)` |
| `F_absdiff` | `(a, b)` | `abs(a - b)` |
| `F_max3` | `(a, b, c)` | `max(a, b, c)` |
| `F_min3` | `(a, b, c)` | `min(a, b, c)` |
| `F_if_gt` | `(a, b, c, d)` | `if a > b then c else d` |
| `F_mod49_sum3` | `(a, b, c)` | `wrap49((a + b + c) % 49)` |

**用户自定义**：`core/function_registry.py::add_function(name, params, body)`。内置函数**不能删**。

**AST 节点**：
```json
{"op": "call_func", "name": "F_sum2",
 "args": [{"factor": "平1", "lag": 1}, {"factor": "平2", "lag": 1}]}
```

**关键安全保证**：如果有人定义的函数**体内藏了 `lag=0` 的因子**（比如 `F_bad(a) = a + 特码(lag=0)`），校验器会**先展开函数再判定**，照样识别为假公式。推导路径里也会先打印 `调用函数 Fxx(...)` 再下钻。

## 🆕 三期交叉模板（底层）

把最近三期的 7 个号码看成 **3×7 矩阵**：

```
         列0=平1  列1=平2  列2=平3  列3=平4  列4=平5  列5=平6  列6=特码
行0 上3期 [  .       .       .       .       .       .       .    ]
行1 上2期 [  .       .       .       .       .       .       .    ]
行2 上1期 [  .       .       .       .       .       .       .    ]
```

支持的方向（`core/cross_templates.py`）：

| 方向 | 函数 | 说明 |
|---|---|---|
| 上下 | `vertical_column(col)` | 同列跨 3 期 |
| 左右 | `horizontal_row(row)` | 同期横向 7 格 |
| 主对角 ↘ | `main_diagonal(start_col)` | 从 `(0, start_col)` 往右下 |
| 反对角 ↙ | `anti_diagonal(start_col)` | 从 `(0, start_col)` 往左下 |
| 十字 | `cross(row, col)` | 中心 + 上下左右（边界自动裁剪） |
| 九宫格 | `neighborhood_3x3(row, col)` | 中心为原点的 3×3，边界自动裁剪 |
| 自定义 | `custom_path(cells)` | 任意点选 2–5 个格子 |

所有方向都会自动做**边界裁剪**。格子列表通过下列函数转成 AST：

- `cells_to_sum_expr(cells)` → `wrap49(sum(...))`
- `cells_to_avg_expr(cells)` → `wrap49(avg(...))`
- `cells_to_max_expr / cells_to_min_expr / cells_to_diff_expr`

因为交叉模板只会取行 0/1/2（分别对应 lag 3/2/1），**所有生成的表达式自动是真公式**。

> 📌 本轮（v3）只交付底层。UI 集成（在 formula_builder 里新增"交叉模板"块、live_predict 显示"用了哪几个格子"、miner 枚举常用方向）会在下一轮完成。

- 找出历史的最后一期（例如 2026/107）
- 下一期 = `(年份, 期数+1)` = 2026/108
- 用最近 3 期（107/106/105）分别作为"上1期/上2期/上3期"代入公式
- 做 `wrap49` 归一到 1~49
- 再映射到目标板块（如"一肖" → 使用 2026 年的生肖表）

**推导路径**逐行打印，例如：
```
1. 预测期号: 2026/108；公式: 映射到生肖[归一到49((上1期.平1 + 上1期.平2))]
2. 上1期(2026/107).平1 = 35
3. 上1期(2026/107).平2 = 10
4. 35 + 10 = 45
5. wrap49(45) = 45
6. 45 → 映射到生肖 → 狗
```

## 🎯 "下一期预测"工作原理

- 找出历史的最后一期（例如 2026/107）
- 下一期 = `(年份, 期数+1)` = 2026/108
- 用最近 3 期（107/106/105）分别作为"上1期/上2期/上3期"代入公式
- 做 `wrap49` 归一到 1~49
- 再映射到目标板块（如"一肖" → 使用 2026 年的生肖表）

**推导路径**逐行打印，例如：
```
1. 预测期号: 2026/108；公式: 映射到生肖[F_sum2(上1期.平1, 上1期.平2)]
2. 调用函数 F_sum2(上1期.平1, 上1期.平2)
3. 上1期(2026/107).平1 = 35
4. 上1期(2026/107).平2 = 10
5. 35 + 10 = 45
6. wrap49(45) = 45
7. 45 → 映射到生肖 → 狗
```

## 📊 新综合评分

```
综合分 = 近100期胜率 × 40%
      + 近50期胜率  × 25%
      + 近20期胜率  × 15%
      + 稳定性       × 10%
      + (1 - 连黑惩罚) × 10%

其中：连黑惩罚 = min(1, 最大连黑 / 20)
      若样本 <30 期，额外乘 (样本数/30) 缩放
```

## 📌 业务口径（不变）

| 口径 | 规则 |
|---|---|
| 大小 | 小 = 1–24，大 = 25–49（**49 算大**） |
| 单双 | 奇数 = 单，偶数 = 双 |
| 合数 | 十位 + 个位。38→11，49→13 |
| 合单双 | 合数奇 = 合单，合数偶 = 合双 |
| 合大合小 | 合 1–6 = 合小，合 7–12 = 合大，**合 13 单独列** |
| 合尾 | 合数的个位 |
| 头数 / 尾数 | 十位 / 个位 |
| 段数 | 1段 1–7，2段 8–14，3段 15–21，4段 22–28，5段 29–35，6段 36–42，7段 43–49 |
| 命中判定 | 全部板块以"特码"所属类别是否等于公式输出为准 |

## 🛠 常见问题

**Q：老公式（v1 的）能继续用吗？**  
A：能。启动后会自动按 `is_predictive` 给所有老公式补上 `predictive` 字段。含 `lag=0` 的老公式会被标为"假公式"，不会出现在实战预测页。

**Q：我想把老公式"修正"成真公式？**  
A：载入到公式构建器，把相关因子的 lag 从"本期"改成"上1期/上2期/上3期"即可。

**Q：怎么重置历史？**  
A：删除 `data/history.csv`，重启即可。

**Q：怎么清空公式库？**  
A：把 `data/formulas.json` 改回 `{"version": 1, "formulas": []}`。
